# 📘 클래스와 인스턴스

## 🎯 오늘 학습 목표
- 클래스와 인스턴스의 개념 차이 명확히 이해하기
- 객체를 생성하고 각각 다른 속성을 가질 수 있음을 확인하기
- 참조형 변수의 동작 방식과 주소 공유 구조 실습하기
- 클래스를 바탕으로 메모리에 객체가 생성되는 흐름 체득하기

---

## 🧠 오늘 배운 개념

### 🔹 클래스 vs 인스턴스
- 클래스는 객체를 만들기 위한 설계도이며, 실체는 아님
- 인스턴스는 클래스를 기반으로 `new` 키워드를 통해 생성된 실제 객체

### 🔹 참조형 변수의 개념
- 객체 자체를 담는 게 아니라, 객체가 있는 **주소**를 저장함
- 참조 변수를 복사하면 **동일한 객체를 공유**하게 됨
- 서로 다른 인스턴스를 만들면 각각 독립적인 상태를 가짐

---

### 🔹 실습 중 사용한 명령어 요약
```java
Car car1 = new Car();          // 인스턴스 생성
car1.color = "red";            // 속성 설정
Car car2 = new Car();          // 또 다른 인스턴스 생성
Car copy = car1;               // 참조값 복사 (같은 객체 가리킴)
```

---

## 🧪 실습 과정 요약
1. Car 클래스를 정의하고, 서로 다른 인스턴스(car1, car2)를 생성하여 속성을 다르게 설정
2. 참조 변수 copy에 기존 인스턴스 original을 복사하여, 같은 객체를 가리키는 구조 실습
3. 값 변경 시 두 변수 모두 동일한 객체의 속성에 영향을 미친다는 점을 출력으로 확인

---

## ⚠️ 실수 & 깨달은 점
- `Car copy = original;`은 객체를 복제하는 게 아니라 주소를 공유하는 것임
- 객체 복사 없이 변수만 복사하면, 하나의 인스턴스를 여러 리모컨으로 조작하는 구조가 됨
- 클래스는 구조를 정의할 뿐이며, new로 인스턴스를 생성해야 비로소 메모리에 실체가 만들어짐

---

## 💭 느낀 점
- 클래스와 인스턴스의 관계를 머리로만 이해했을 땐 헷갈렸는데, 직접 객체를 만들고 비교하니 이미지처럼 떠오르게 됐다.
- 참조 변수의 복사가 주소 공유라는 걸 실습으로 직접 확인하고 나니, 객체 조작 구조가 훨씬 명확해졌다.
- 앞으로 객체를 만들 땐 설계도와 실체를 구분해서 생각하는 습관이 필요하다는 걸 느꼈다.
